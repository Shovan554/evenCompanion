# even-reminder (Mac App) Implementation Plan

> Executed via subagent-driven development. Tasks are design-level contracts; implementers write Swift TDD-style (mid-tier model). Steps use checkbox syntax.

**Goal:** Build the native macOS menu bar app `even-reminder` — the always-on hub that collects Mac stats + Reminders and publishes 1s snapshots to the relay, with two menu bar items (Reminders + Settings/Control).

**Architecture:** A Swift Package with a testable library target `EvenReminderCore` and a SwiftUI executable target `EvenReminder`. All telemetry is gathered by running standard macOS CLI tools and parsing their text output via PURE parser functions (unit-tested with canned fixtures); a thin injected `CommandRunner` executes the commands so the builder is testable. The snapshot encodes to the EXACT JSON the relay forwards and the glasses `parseSnapshot` accepts. EventKit (Reminders) and the relay WebSocket sit behind protocols. The SwiftUI layer renders two `MenuBarExtra` scenes and drives a 1s publish timer.

**Tech Stack:** Swift 6 / SwiftPM, SwiftUI + AppKit (`MenuBarExtra`), EventKit, `URLSessionWebSocketTask`, `SMAppService` (launch at login). Lives in `mac/`. Verified with `swift build` + `swift test` (host macOS).

## Global Constraints

- All code under `mac/` as a SwiftPM package named `EvenReminder`. Two targets: `EvenReminderCore` (library, pure-ish, fully unit-tested) and `EvenReminder` (executable, SwiftUI app) depending on Core; plus test target `EvenReminderCoreTests`.
- **The published snapshot JSON MUST exactly match the data contract** (design spec §4) so the glasses app's deep `parseSnapshot` accepts it. Keys & nesting:
  `{ ts:Int, host:String, vitals:{ cpu:Double, ramUsedGB:Double, ramTotalGB:Double, ramPressure:String, diskFreeGB:Double, diskTotalGB:Double, ssdHealth:String, ssdWearPct:Double|null }, netProc:{ upKBs:Double, downKBs:Double, topProc:{name:String,cpu:Double}|null, battery:{pct:Int,charging:Bool}|null }, ports:[{port:Int,proc:String}], reminders:[{id:String,title:String,due:Int|null,overdue:Bool}] }`.
  (Encode `null` for absent optionals via Swift optionals + `encodeIfPresent`/explicit null — the glasses guard treats missing vs explicit null the same for `battery`/`topProc`/`ssdWearPct`/`due`, but match the contract: those four may be null; all other fields are required.)
- Telemetry via these commands (parse their output — do not use private APIs): CPU `top -l 2 -n 0` (use the 2nd "CPU usage" line for a real interval), memory `vm_stat` + `sysctl -n hw.memsize`, disk `df -k /`, SSD health `diskutil info /` (SMART Status), ports `lsof -nP -iTCP -sTCP:LISTEN`, network bytes `netstat -ib`, battery `pmset -g batt`, top process `ps -Aceo pcpu,comm -r`.
- Network throughput = delta of interface bytes between two samples / elapsed seconds → KB/s. The builder keeps the previous sample.
- Reverse command handling: on receiving `{cmd:"completeReminder", id}` from the relay, complete that reminder via EventKit.
- Reminders published = upcoming incomplete reminders (next ~10), each `due` = epoch seconds or null, `overdue` = due in the past.
- Config persisted in `UserDefaults`: `relayUrl` (e.g. `wss://maccompanion-relay.onrender.com`), `relayToken`, `publishEnabled` (Bool), per-screen enable toggles. Secret token acceptable in UserDefaults for v1 (note: Keychain is a v2 hardening).
- App is a menu bar agent: `LSUIElement`/`.accessory` activation (no Dock icon). Provide an app icon asset placeholder.

## File Structure (`mac/`)
- `Package.swift` — two targets + test target, macOS 14+ platform.
- `Sources/EvenReminderCore/Snapshot.swift` — Codable models + a sample encoder; `snapshotJSON(_:) -> String`.
- `Sources/EvenReminderCore/CommandRunner.swift` — `protocol CommandRunner { func run(_ cmd: String, _ args: [String]) -> String }` + `SystemCommandRunner` (Process-based) + the parser-friendly seam.
- `Sources/EvenReminderCore/Parsers.swift` — pure funcs: `parseTopCPU`, `parseVmStat`, `parseDf`, `parseDiskutilSMART`, `parseLsofListening`, `parseNetstat`, `parsePmsetBattery`, `parseTopProcess`. Each takes raw text → typed value(s).
- `Sources/EvenReminderCore/SnapshotBuilder.swift` — injects a `CommandRunner`; `build(now:host:prev:) -> (Snapshot, NetSample)`; computes net rate from prev sample.
- `Sources/EvenReminderCore/RemindersStore.swift` — `protocol RemindersProviding { func upcoming() async -> [Reminder]; func complete(id:) async }` + `EventKitRemindersStore`.
- `Sources/EvenReminderCore/RelayPublisher.swift` — `protocol RelayPublishing` + `WebSocketRelayPublisher` (URLSessionWebSocketTask, role=pub, reconnect, onCommand callback).
- `Sources/EvenReminderCore/Config.swift` — `AppConfig` (UserDefaults-backed observable settings).
- `Sources/EvenReminder/EvenReminderApp.swift` — `@main`, two `MenuBarExtra` scenes.
- `Sources/EvenReminder/AppModel.swift` — `@MainActor ObservableObject`: 1s timer → build snapshot → publish; holds reminders; applies reverse commands.
- `Sources/EvenReminder/RemindersView.swift`, `SettingsView.swift` — the two menu bar UIs.
- `Sources/EvenReminder/LoginItem.swift` — `SMAppService` register/unregister.
- `Sources/EvenReminder/Resources/` — `Info.plist` (LSUIElement true), app icon placeholder.
- `Tests/EvenReminderCoreTests/*.swift` — parser fixtures, snapshot JSON contract, builder with a fake runner.

### Task M1: SwiftPM scaffold + Snapshot model + JSON contract test
**Deliverable:** package builds; `Snapshot` encodes to the exact contract JSON.
- [ ] `mac/Package.swift` with `EvenReminderCore` (library), `EvenReminder` (executable, depends on Core), `EvenReminderCoreTests`. Platforms `.macOS(.v14)`. Swift tools 6.0.
- [ ] `Snapshot.swift`: `Codable` structs `Snapshot, Vitals, NetProc, TopProc, Battery, Port, Reminder` with property names matching the contract EXACTLY (Swift props == JSON keys; use them directly, no snake_case). Optionals: `topProc`, `battery`, `ssdWearPct`, `due` are `Optional` and MUST encode as JSON `null` when nil (use a JSONEncoder; for explicit null on optionals, encode via `encode(_:forKey:)` in custom `encode` OR rely on `encodeIfPresent` — BUT the contract wants null present; implement custom `encode(to:)` that writes null for those four). `host` and all vitals/netProc-number fields required.
- [ ] A temporary `Sources/EvenReminder/main.swift` or minimal `@main` so the executable target compiles (M4 builds the real app). For M1 a stub `print("even-reminder")` executable is fine.
- [ ] TDD `Tests/.../SnapshotContractTests.swift`: build a sample `Snapshot`, encode with `JSONEncoder` (sortedKeys), assert the decoded-back dictionary has exactly the contract keys at each level, that `battery`/`topProc`/`ssdWearPct`/`due` serialize as JSON `null` when nil, and that a populated one round-trips. (This is the cross-app contract guard.)
- [ ] `cd mac && swift build` and `swift test` green. Commit `feat(mac): swiftpm scaffold + snapshot contract model`.

### Task M2: Stat collectors (pure parsers) + SnapshotBuilder
**Deliverable:** every telemetry parser unit-tested against canned fixtures; builder assembles a Snapshot from a fake runner.
- [ ] TDD `Parsers.swift` — implement each parser with a realistic fixture captured in the test:
  - `parseTopCPU(_:) -> Double` (sum of user+sys from the CPU usage line).
  - `parseVmStat(_:pageSize:memTotalBytes:) -> (usedGB:Double, pressure:String)` (used = active+wired+compressed pages).
  - `parseDf(_:) -> (freeGB:Double, totalGB:Double)`.
  - `parseDiskutilSMART(_:) -> String` (e.g. "Verified").
  - `parseLsofListening(_:) -> [Port]` (dedupe by port; proc = command name).
  - `parseNetstat(_:) -> (inBytes:UInt64, outBytes:UInt64)` (sum across non-loopback interfaces).
  - `parsePmsetBattery(_:) -> Battery?` (nil if desktop/no battery).
  - `parseTopProcess(_:) -> TopProc?` (highest %cpu row, skipping the header).
  Each gets ≥1 fixture-based test asserting exact parsed values, plus an empty/garbage-input test returning a safe default/nil.
- [ ] TDD `SnapshotBuilder.swift`: `struct NetSample { let inBytes; let outBytes; let at: Double }`. `build(now:Double, host:String, prev:NetSample?) -> (Snapshot, NetSample)` runs the commands via the injected `CommandRunner`, parses, computes net KB/s from `prev` (0 if no prev), assembles the `Snapshot`. Test with a `FakeCommandRunner` returning canned outputs keyed by command → assert the assembled Snapshot fields and that a second build with a prev sample yields a positive throughput.
- [ ] `swift build` + `swift test` green. Commit `feat(mac): telemetry parsers + snapshot builder`.

### Task M3: EventKit reminders + relay publisher + config
**Deliverable:** reminders + relay abstractions with protocols; config persistence; tests where no system permission is needed.
- [ ] `RemindersStore.swift`: `struct ReminderDTO`/reuse `Reminder`; `protocol RemindersProviding { func requestAccess() async -> Bool; func upcoming(limit:Int) async -> [Reminder]; func complete(id:String) async }`. `EventKitRemindersStore` implements it with `EKEventStore` (map `EKReminder` → `Reminder`: id = `calendarItemIdentifier`, due = dueDateComponents→epoch, overdue = due<now). Provide an in-memory `FakeRemindersStore` in the test target. TDD: `FakeRemindersStore.complete` marks completed and removes from `upcoming`; overdue mapping logic if extracted as a pure helper `isOverdue(due:now:)`.
- [ ] `RelayPublisher.swift`: `protocol RelayPublishing { func start(); func publish(_ json:String); func stop(); var onCommand: ((String)->Void)? { get set } }`. `WebSocketRelayPublisher` uses `URLSessionWebSocketTask` to `\(url)/ws?token=\(token)&role=pub`, sends snapshot JSON, receives text frames → `onCommand`, reconnects with backoff. (No unit test of the live socket; keep logic minimal. If any pure helper exists — e.g. `publisherURL(base:token:) -> URL` — unit-test that, including token percent-encoding.)
- [ ] `Config.swift`: `AppConfig` reading/writing `relayUrl`, `relayToken`, `publishEnabled`, screen toggles via `UserDefaults` (injectable `UserDefaults` for tests). TDD round-trip: set values, read back; defaults when unset.
- [ ] `swift build` + `swift test` green. Commit `feat(mac): eventkit reminders, relay publisher, config`.

## Notes / verification limits
- EventKit, the live WebSocket, `SMAppService`, and real telemetry require user permission/hardware and are validated when the user runs the app — not in `swift test`. The pure parsers, snapshot contract, builder (fake runner), config, and fake reminders ARE tested. `swift build` proves the whole thing (incl. SwiftUI app) compiles.
- Packaging into a signed `.app` with the icon and shipping login-item is a documented post-build step (covered in STATUS / README).
